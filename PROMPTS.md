### Prompt: Database Design Illustration

I have the following database design model:

```text
{{data}}
```

Please create a professional, high-quality database design diagram suitable for technical documentation.

### Requirements

* Illustrate the complete database schema in a clean ERD-style layout.
* Represent each table as a structured table containing:

  * Column Name
  * Data Type
  * Constraints (Primary Key, Foreign Key, Unique, Not Null, Default, etc.)
  * Description
* Clearly identify all **Primary Keys (PK)** and **Foreign Keys (FK)**.
* Draw relationship lines between all related tables.
* Label each relationship with its correct cardinality (One-to-One, One-to-Many, Many-to-One, or Many-to-Many where applicable).
* Visually distinguish each table using a modern, professional color palette while maintaining readability.
* Include a legend explaining all icons and symbols used in the diagram.
* Add a section summarizing:

  * Table relationships
  * Database constraints
  * Important business rules
* Use database-specific icons for keys, foreign keys, timestamps, numeric values, text fields, enums, and other common database concepts.
* Ensure all text is grammatically correct and professionally formatted.
* Maintain consistent spacing, alignment, typography, and connector lines.
* Use a clean white background suitable for inclusion in project documentation and GitHub README files.
* The final output should resemble a professionally designed database architecture diagram created by a senior software engineer.
